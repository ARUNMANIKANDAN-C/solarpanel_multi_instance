🔬 FastViT vs MobileViTv2

Comparative Study of Efficient Architectures for Solar Panel Multi-Defect Detection

Project Overview

This research repository implements a modular Faster R-CNN pipeline comparing two efficient architectures: FastViT-T8 (30M parameters, CNN-based) and MobileViTv2-100 (21.7M parameters, CNN+Transformer hybrid) for detecting solar panel defects on the PV-Multi-Defect dataset.

Object Detection Solar Panels Faster R-CNN Efficient Architectures

📊 Dataset & Preprocessing

Aspect Details
Images 1,106 images with 3,981 annotated objects
Defect Classes Scratch, Black Border, No Electricity, Hot Spot, Broken
Augmentation FixedAugmenter: flip, brightness, contrast, noise, blur, HSV, rotation, scaling, translation, cutout (11 techniques)
Target Balance 9,000 samples per class (augmentation-based balancing)
Train/Val/Test 70% / 15% / 15% split
Training Config 200 epochs, batch_size=16, lr=1e-4, AdamW optimizer

🏗️ Architecture Comparison

FastViT-T8

  • Type: Hybrid CNN + Structural Re-parameterization
  • Params: 29.97M (26.8M trainable)
  • Model Size: 16.3 MB
  • FPN: 4-level standard FPN
  • Detail Block: 6-branch (k=7,9,11,13,H,V)
  • Attention: CoordAtt + SE + CBAM
  • Anchors: Single per level
  • Class Balance: Via augmentation
  • Unfreezing: Gradual (25 epochs)

MobileViTv2-100

  • Type: Hybrid CNN + Transformer (Linear Attention)
  • Params: 21.70M (17.3M trainable)
  • Model Size: 19.7 MB
  • FPN: 3-level BiFPN with parallel branches
  • Detail Block: 3-branch (k=7,11,13)
  • Attention: Spatial + CBAM + WeightedAdd
  • Anchors: Paired per level (extreme AR)
  • Class Balance: WeightedRoIHeads + loss weighting
  • Unfreezing: Fast (20 epochs)

🚀 Quick Start

  1. Install dependencies:
    pip install -r requirements.txt
  2. Run FastViT notebook:
    jupyter notebook base-template(1).ipynb
  3. Or run MobileViTv2 notebook:
    jupyter notebook base-template-mobilevit.ipynb
  4. Or run via command line:
    python main.py --epochs 200 --batch_size 16

📈 Key Research Insights

  • Efficient Architectures: Both models achieve <200 epochs with <30M parameters for real-time solar panel inspection on edge devices.
  • Architecture Trade-offs: FastViT prioritizes rich local feature extraction; MobileViTv2 adds global context via transformers.
  • Class Imbalance: FastViT relies on augmentation; MobileViTv2 uses explicit class weighting for severe imbalance (hot_spot overrepresented).
  • Transfer Learning: Stage-wise unfreezing enables effective fine-tuning from ImageNet pretraining.
  • Modular Design: Shared preprocessing pipeline enables fair architectural comparison.
